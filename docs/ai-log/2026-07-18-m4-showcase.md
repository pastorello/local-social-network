## Decisions

1. **Product name: "Gaeta Partecipa"**, repo/package names stay `local-social-network`
   (author call, recorded in spec + CLAUDE.md). Rebrand touches `SiteTitle.vue`,
   `index.html` (`lang="it"`, title, meta description) — no backend surface changes.
2. **Deployment: prep only.** gunicorn + whitenoise are wired and correct for either
   a PaaS or a VPS; `docs/deploy.md` sketches both without committing to one. The
   dev `docker-compose.yml` is explicitly *not* claimed production-ready.
3. **JWT hardening lands now, not later**: access 30d→**60min**, refresh
   180d→**30 days**, `ROTATE_REFRESH_TOKENS` + `BLACKLIST_AFTER_ROTATION` on. A
   60-minute access token is useless without transparent renewal, so the SPA gets
   a 401-response interceptor (`lib/auth.ts`) — single-flight (concurrent 401s
   share one refresh call), one retry per request, skips `/login/` and `/refresh/`
   themselves to avoid loops.
4. **`user.ts` `refreshToken()` becomes async and persists the rotated refresh
   token** (previously it only touched the access token — fine when rotation was
   off, wrong now: the old refresh token is blacklisted server-side the moment a
   new one is issued, so failing to persist it would strand the session on the
   next boot).
5. **PR #4 (M3 polls) had conflicts against `main`** — `m3-polls` had been built on
   top of `m2-map-reports`, which was squash-merged rather than fast-forwarded.
   Fixed by rebasing `m3-polls` onto `origin/main` (`git rebase --onto origin/main
   m2-map-reports m3-polls`), which dropped the now-duplicate M2 commits and kept
   only the 5 genuine M3 commits; force-pushed. `m4-showcase` was re-pointed at
   the rebased history (it had no commits of its own yet). **Follow-up for the
   author:** delete `m3-polls` after merging PR #4 so future stacked PRs retarget
   `main` automatically.
6. **Login redirect honors `?redirect=`**: the router guard already set the query
   param on an auth bounce, but `LoginView` ignored it and always pushed `/`. Fixed
   — one line, but it's the actual UX bug behind "why did I land on the map after
   logging in from a gated page."
7. **`self-start` layout fix extended to `HomeView` and `ReportDetailView`** (the
   `PanelBox is h-full` stretch bug fixed only in `PollsView` during M3 — flagged
   as latent elsewhere in that session's review notes). `UsersView`'s dead "Right
   column" placeholder panel is removed instead of translated — nothing renders
   there and it was tutorial-era filler.

## Changes

### Backend
- `SIMPLE_JWT`: `ACCESS_TOKEN_LIFETIME=60min`, `REFRESH_TOKEN_LIFETIME=30 days`,
  `ROTATE_REFRESH_TOKENS=True`, `BLACKLIST_AFTER_ROTATION=True`;
  `rest_framework_simplejwt.token_blacklist` installed (migrations applied).
- Production serving: `whitenoise` middleware + `STATIC_ROOT`/`collectstatic`
  (`STORAGES['staticfiles']` swaps to `CompressedManifestStaticFilesStorage` only
  when `DEBUG=false`, so bare `runserver` never needs `collectstatic`); `gunicorn`
  pinned in `requirements.txt`.
- HTTPS hardening block (`SECURE_SSL_REDIRECT`, `SESSION_COOKIE_SECURE`,
  `CSRF_COOKIE_SECURE`, `SECURE_HSTS_SECONDS`) — active only when `DJANGO_DEBUG=false`;
  `DJANGO_SSL_REDIRECT` env var to disable the redirect specifically (proxy-terminated TLS).
- 1 new test: refresh rotates the token and blacklists the old one (replay → 401).
  Backend total: **76**.

### Frontend
- Rebrand: `SiteTitle.vue`, `index.html`.
- `lib/auth.ts` (new): axios response interceptor, single-flight refresh, one
  retry, skips auth endpoints. Wired in `main.ts` right after `pinia` (needs the
  user store) and before `router`.
- `stores/user.ts`: `refreshToken()` is now `async`, returns whether it succeeded,
  and persists a rotated refresh token when the backend sends one.
- Italian copy sweep: `AppHeader`, `LoginView`, `SignupView`, `UsersView`,
  `EditProfileView`, `EditPasswordView` (labels, buttons, validation/error
  messages — the codebase had drifted to a mix of English scaffolding and Italian
  product copy since M1/M2).
- `LoginView` reads `route.query.redirect` and pushes it after a successful login
  instead of always going to `/`.
- Layout: `self-start` added to `HomeView` and `ReportDetailView`'s stacked
  columns (same PanelBox `h-full` issue fixed for `PollsView` in M3).
- `UsersView`: dead "Right column" panel removed, left column now spans the full
  width (4 cols instead of 3+1), user grid goes to 4 columns.
- 8 new/updated tests: `lib/auth.spec.ts` (interceptor: refresh+retry, skips
  login/refresh 401s, gives up and clears session on failed refresh, retries only
  once), 2 new cases in `stores/user.spec.ts` (rotation persisted, session cleared
  on refresh failure), `LoginView`/`SignupView` specs updated for the Italian copy.
  Frontend total: **32**.

`/api/users/list/` succeeds) — same rotation-persistence code path the
401-interceptor's proactive sibling uses. Checked layout: `HomeView`'s left
